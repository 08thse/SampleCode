1. Oracle WebLogic Server 11g 製品概要

2. 基本機能
・MAN/WAN レプリケーション
　- EJB コンテナのリモート呼び出しはオーバーヘッドが大きくアンチパターン化している

・RMI
　- リモートスタブ：呼び出し側
　- リモートスケルトン：呼び出される側

今さら人に訊けない JNDI 第 1 回 「JNDI ってなんだっけ？」
http://www.ibm.com/developerworks/jp/websphere/library/was/was_jndi/1.html#ibm-pcon

・JDBC
　- 設定は weblogic-jdbc.xsd という専用 XML ファイルに記述する

3. 構築
・Apache との連携は、mod_wl_**.so モジュールを LoadModule して、SetHandler/PathTrim や 
IfModule ディレクティブの設定を行えば OK

4. 応用コンフィグレーション
・JMS：送信側をプロデューサ、受信側をコンシューマと呼ぶ
・DistributedQueue で負荷分散など。共通分散送り先の利用を推奨
・WebLogic JMS C API：C 言語プログラムから JNI を利用して、クライアントサイドの Java 
JMS クラスにアクセス
・セキュリティレルム：要はユーザ／ロール管理
・ノードマネージャ：管理サーバや管理対象サーバの起動／停止やログ取得などが可能
・JMS や JTA 関連のサービスでシングルトンサービスになっているものは、障害時に
永続ストアの滞留データを復旧させる必要がある。これに「移行」が利用できる。
・サーバー自動移行の概要と設定。復旧後の切り戻しは手動で実施する
・運用に便利な WebLogic 診断フレームワーク (WLDF)、SNMP
